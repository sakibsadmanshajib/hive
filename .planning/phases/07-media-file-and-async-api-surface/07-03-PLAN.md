Implement the Files API, Uploads API, and Batches API with an async batch worker for upstream provider polling.

Output: Working /v1/files/*, /v1/uploads/*, and /v1/batches/* endpoints in edge-api, plus an Asynq-based batch worker in control-plane that polls upstream providers and assembles results.

`ServeHTTP` dispatches by method + path pattern:
- `POST /v1/files` -> `handleUploadFile`
- `GET /v1/files` (no trailing path) -> `handleListFiles`
- `GET /v1/files/{id}` (no `/content` suffix) -> `handleRetrieveFile`
- `GET /v1/files/{id}/content` -> `handleFileContent`
- `DELETE /v1/files/{id}` -> `handleDeleteFile`
- `POST /v1/uploads` -> `handleCreateUpload`
- `GET /v1/uploads/{id}` -> `handleRetrieveUpload`
- `POST /v1/uploads/{id}/parts` -> `handleAddPart`
- `POST /v1/uploads/{id}/complete` -> `handleCompleteUpload`
- `POST /v1/uploads/{id}/cancel` -> `handleCancelUpload`

Parse `{id}` from path using `strings.TrimPrefix` and `strings.Split`.

`handleUploadFile(w, r)`:
1. Authorize via `h.authorizer.Authorize(r)` to get `AuthSnapshot`
2. Parse multipart: `r.ParseMultipartForm(MaxFileSize)`
3. Extract `file` and `purpose` from form
4. Validate purpose against `ValidPurposes` map; return 400 if invalid
5. Validate file size <= `MaxFileSize`; return 400 if too large
6. Generate file ID: `file-{uuid}`
7. Compute storage path: `{accountID}/{fileID}/{filename}`
8. Upload blob to S3: `h.storage.Upload(ctx, h.bucket, storagePath, file, size, contentType)`
9. Create metadata: `h.fileClient.CreateFile(ctx, accountID, purpose, filename, size, storagePath)`
10. Return `FileObject` JSON with status 200

`handleListFiles(w, r)`:
1. Authorize
2. Extract optional `purpose` query param
3. Call `h.fileClient.ListFiles(ctx, accountID, purpose)`
4. Return `FileListResponse` JSON

`handleRetrieveFile(w, r)`:
1. Authorize
2. Parse file ID from path
3. Call `h.fileClient.GetFile(ctx, fileID, accountID)` -- account_id enforced server-side
4. Return `FileObject` JSON or 404 if not found / not owned

`handleFileContent(w, r)`:
1. Authorize
2. Parse file ID from path
3. Get file metadata (to obtain storage_path): `h.fileClient.GetFile(ctx, fileID, accountID)`
4. Download from S3: `reader, err := h.storage.Download(ctx, h.bucket, file.StoragePath)`
5. Set `Content-Type` based on filename extension (or `application/octet-stream`)
6. `io.Copy(w, reader)`

`handleDeleteFile(w, r)`:
1. Authorize
2. Parse file ID from path
3. Get file metadata for storage_path
4. Delete blob: `h.storage.Delete(ctx, h.bucket, file.StoragePath)`
5. Delete metadata: `h.fileClient.DeleteFile(ctx, fileID, accountID)`
6. Return `DeletedFileResponse{ID: fileID, Object: "file", Deleted: true}`

`handleCreateUpload(w, r)`:
1. Authorize
2. Read JSON body: `{filename, purpose, bytes, mime_type}`
3. Validate purpose and bytes <= MaxFileSize
4. Generate upload ID: `upload-{uuid}`
5. Compute storage path: `{accountID}/{uploadID}/{filename}`
6. Init S3 multipart: `s3UploadID, err := h.storage.InitMultipartUpload(ctx, h.bucket, storagePath, mimeType)`
7. Create metadata: `h.fileClient.CreateUpload(ctx, accountID, filename, bytes, mimeType, purpose)` -- include s3_upload_id
8. Return `UploadObject` JSON

`handleAddPart(w, r)`:
1. Authorize
2. Parse upload ID from path
3. Get upload metadata to verify ownership and get s3_upload_id + storage_path
4. Parse multipart form, extract `data` file part
5. Upload part: `partInfo, err := h.storage.UploadPart(ctx, h.bucket, storagePath, s3UploadID, partNum, data, size)`
6. Generate part ID: `part-{uuid}`
7. Record part: `h.fileClient.AddUploadPart(ctx, uploadID, partID, partNum, partInfo.ETag)`
8. Return `UploadPartObject` JSON

`handleCompleteUpload(w, r)`:
1. Authorize
2. Parse upload ID from path
3. Read JSON body: `{part_ids: []string}`
4. Get upload metadata + list parts ordered by part_num
5. Build `[]old storage client CompletePart` from parts
6. Complete S3 multipart: `h.storage.CompleteMultipartUpload(ctx, h.bucket, storagePath, s3UploadID, completeParts)`
7. Complete upload metadata (creates File record): `h.fileClient.CompleteUpload(ctx, uploadID, accountID, partIDs)`
8. Return `UploadObject` with status="completed" and file field

`handleCancelUpload(w, r)`:
1. Authorize
2. Parse upload ID from path
3. Get upload metadata
4. Abort S3 multipart: `h.storage.AbortMultipartUpload(ctx, h.bucket, storagePath, s3UploadID)`
5. Cancel metadata: `h.fileClient.CancelUpload(ctx, uploadID, accountID)`
6. Return `UploadObject` with status="cancelled"

- `POST /v1/batches/{id}/cancel` -> `handleCancel`

`handleCreate(w, r)`:
1. Authorize
2. Read JSON body as `BatchRequest`
3. Validate `endpoint` against `ValidBatchEndpoints`; return 400 if invalid
4. Validate `completion_window` is "24h" (only supported value); return 400 otherwise
5. **Validate JSONL before reserving credits:**
   - Get input file metadata via `h.fileClient.GetFile(ctx, inputFileID, accountID)` (validates ownership)
   - Download file content: `reader, err := h.storage.Download(ctx, h.bucket, file.StoragePath)`
   - Scan line-by-line with `bufio.Scanner`:
     - Parse each line as JSON into `BatchInputLine`
     - Validate `custom_id` is non-empty
     - Validate `method` is "POST"
     - Validate `url` matches batch `endpoint`
     - Count total lines
   - If any line fails validation, return 400 with error details (line number, reason) BEFORE any credit reservation
6. Estimate credit cost from line count and endpoint type
7. Reserve credits via `h.accounting` client
8. Create batch: `h.batchClient.CreateBatch(ctx, accountID, inputFileID, endpoint, completionWindow, lineCount, reservationID)`
9. Return `BatchObject` JSON

`handleList(w, r)`:
1. Authorize
2. Parse `limit` (default 20, max 100) and `after` query params
3. Call `h.batchClient.ListBatches(ctx, accountID, limit, after)`
4. Return `BatchListResponse` JSON

`handleRetrieve(w, r)`:
1. Authorize
2. Parse batch ID from path
3. Call `h.batchClient.GetBatch(ctx, batchID, accountID)`
4. Return `BatchObject` JSON or 404

`handleCancel(w, r)`:
1. Authorize
2. Parse batch ID from path
3. Call `h.batchClient.CancelBatch(ctx, batchID, accountID)`
4. Return updated `BatchObject` JSON

`HandleBatchPoll(ctx context.Context, t *asynq.Task) error`:
1. Unmarshal payload to `BatchPollPayload`
2. Check upstream batch status:
   - GET `{litellmBaseURL}/v1/batches/{upstreamBatchID}` with Authorization header
   - Parse response to get status
3. Switch on upstream status:
   - **`completed`**:
     a. Download output file from upstream: GET `{litellmBaseURL}/v1/files/{outputFileID}/content`
     b. Generate output file ID: `file-{uuid}`
     c. Upload output JSONL to S3: `storageClient.Upload(ctx, bucket, storagePath, reader, size, "application/jsonl")`
     d. Create file metadata via `fileService.CreateFile(...)` with purpose="batch_output"
     e. If error file exists, repeat for error file with purpose="batch_output"
     f. Update batch status to "completed" with output_file_id, completed_at, final request_counts
     g. Finalize credit reservation with actual usage from response
     h. Return nil (task complete)
   - **`in_progress`**:
     a. Update batch request_counts in Postgres if changed
     b. Return `fmt.Errorf("batch still in progress")` -- Asynq will retry with configured delay
   - **`failed`**:
     a. Update batch status to "failed" with failed_at
     b. Release credit reservation (refund)
     c. Return nil (task complete)
   - **`cancelled`**:
     a. Update batch status to "cancelled" with cancelled_at
     b. Release credit reservation (refund)
     c. Return nil
