// ImageEdit forwards a pre-built multipart request to /images/edits
func (c *LiteLLMClient) ImageEditRaw(ctx context.Context, body io.Reader, contentType string) (*http.Response, error) {
    // Build request manually (not using dispatch which assumes JSON body):
    url := c.baseURL + "/images/edits"
    req, err := http.NewRequestWithContext(ctx, http.MethodPost, url, body)
    req.Header.Set("Content-Type", contentType)
    req.Header.Set("Authorization", "Bearer "+c.masterKey)
    return c.client.Do(req)
}

`ServeHTTP` dispatches by path:
- `/v1/images/generations` -> `handleGeneration`
- `/v1/images/edits` -> `handleEdit`
- `/v1/images/variations` -> return 501 unsupported error via `errors.WriteError(w, 501, "invalid_request_error", "Image variations are not supported. Use generations or edits instead.", "unsupported_operation")`
- anything else -> 404

`handleGeneration(w, r)`:
1. Authorize request via `h.authorizer.Authorize(r)` to get `AuthSnapshot`
2. Read JSON body, unmarshal to `ImageGenerationRequest`
3. Call routing client with `NeedImageGeneration: true` and the model alias
4. Reserve credits via accounting client (estimated cost based on model + size + n)
5. Rewrite model in body to `litellmModel` from route result
6. Dispatch via `h.litellm.ImageGeneration(ctx, litellmModel, rewrittenBody)`
7. Read upstream response JSON into `ImageResponse`
8. Normalize response:
   - Replace model name with Hive alias in any model field
   - If `response_format == "url"` (default): for each `data[i]`, download the image bytes from the provider URL, upload to S3 at `images/{account_id}/{uuid}.png`, generate presigned URL with 1-hour TTL, replace `data[i].URL` with the signed URL
   - If `response_format == "b64_json"`: pass through `b64_json` field as-is
9. Finalize credits via accounting client
10. Write response JSON to client

`handleEdit(w, r)`:
1. Authorize request
2. Parse multipart form: `r.ParseMultipartForm(32 << 20)` (32MB limit for edits)
3. Extract model from form value, route with `NeedImageEdit: true`
4. Reserve credits
5. Rebuild multipart body using `io.Pipe()` + `multipart.NewWriter`:
   - Copy all form fields from original request
   - Replace `model` field value with `litellmModel`
   - Stream file parts (image, mask) from original to new multipart
6. Forward via `h.litellm.ImageEditRaw(ctx, pipeReader, newMultipartWriter.FormDataContentType())`
7. Normalize response same as generation (signed URL replacement for url mode)
8. Finalize credits
9. Write response JSON

`ServeHTTP` dispatches by path:

- `/v1/audio/transcriptions` -> `handleTranscription`
- `/v1/audio/translations` -> `handleTranslation`
- anything else -> 404

`handleSpeech(w, r)`:
1. Authorize via `h.authorizer.Authorize(r)`
2. Read JSON body, unmarshal to `SpeechRequest`
3. Route with `NeedTTS: true` and model alias
4. Reserve credits -- estimate based on character count: `len([]rune(request.Input))` characters
5. Rewrite model in body to `litellmModel`
6. Dispatch: `resp, err := h.litellm.Speech(ctx, litellmModel, rewrittenBody)`
7. **Binary relay** (no JSON wrapping):
   - Copy `Content-Type` header from upstream response directly: `w.Header().Set("Content-Type", resp.Header.Get("Content-Type"))`
   - Copy `Content-Length` if present
   - `w.WriteHeader(http.StatusOK)`
   - `io.Copy(w, resp.Body)` -- pipe binary directly, no buffering
   - `resp.Body.Close()`
8. Finalize credits with character-count-based metering

`handleTranscription(w, r)`:
1. Authorize
2. Parse multipart form: `r.ParseMultipartForm(25 << 20)` (25MB for audio)
3. Extract model from form value, route with `NeedSTT: true`
4. Reserve credits (estimate from file size; refine after getting duration from response)
5. Rebuild multipart body using `io.Pipe()` + `multipart.NewWriter`:
   - Copy all form fields (language, response_format, temperature, timestamp_granularities)
   - Replace `model` field with `litellmModel`
   - Stream the audio file part directly from original to new multipart (NEVER write to disk or storage -- PRIV-01)
6. Forward: `resp, err := h.litellm.TranscriptionRaw(ctx, pipeReader, mw.FormDataContentType())`
7. Read response JSON, extract `duration` field for metering
8. Finalize credits with duration-based metering (seconds of audio)
9. Write response JSON to client (pass through as-is from LiteLLM)

`handleTranslation(w, r)`:
- Same pattern as transcription but routes with `NeedSTT: true` (translation uses same whisper model)
- Forwards to `/audio/translations` via `h.litellm.TranslationRaw`
- Same duration-based metering
