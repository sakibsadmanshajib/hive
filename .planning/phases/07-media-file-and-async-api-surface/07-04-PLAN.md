<objective>
Close the three verification gaps (authorization, routing, accounting) in the images and audio handlers.

Purpose: Plans 07-02 shipped image and audio handlers with intentionally simplified LiteLLM dispatch. Verification found that both handlers bypass auth, routing, and accounting — meaning unauthenticated callers can generate images and audio without credit charges, and model names pass through to LiteLLM without capability-based route selection. This plan adds the full orchestrator lifecycle (authorize -> route -> reserve -> dispatch -> finalize) to both handlers, matching the pattern already established in the batches handler.

Output: Both images and audio handlers enforce API key auth, use routing client for provider selection, and track credit usage via accounting client. The 3 failed verification truths become passing.
</objective>

```go
func NewAuthorizerAdapter(inner *authz.Authorizer) *AuthorizerAdapter {
	return &AuthorizerAdapter{inner: inner}
}

func (a *AuthorizerAdapter) AuthorizeRequest(r *http.Request) (AuthResult, error) {
	authHeader := r.Header.Get("Authorization")
	snapshot, _, authErr := a.inner.Authorize(r.Context(), authHeader, "", 0, 0, 0)
	if authErr != nil {
		return AuthResult{}, fmt.Errorf("unauthorized: %s", authErr.Error.Message)
	}
	return AuthResult{
		AccountID: snapshot.AccountID,
		APIKeyID:  snapshot.KeyID,
	}, nil
}
```
