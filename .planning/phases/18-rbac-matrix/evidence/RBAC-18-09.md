# RBAC-18-09 — vitest matrix-parity: FE decisions == Go decisions for every (actor, perm) tuple

## Scope

`apps/web-console/tests/unit/permissions.parity.test.ts` contains a hardcoded 55-cell
fixture derived from `TestPolicyMatrix` in `policy_test.go`. For every `(actor, perm)`
tuple, the vitest test asserts that `can(viewer, perm)` returns the same boolean as
the Go `Policy.Can` decision. Two size guards (`PERMISSIONS.length===11`,
`MATRIX.length===5`) ensure the fixture stays in sync with the registry. CI's
codegen-drift check (`make gen-permissions && git diff --exit-code`) catches any
Go-side registry change that was not reflected in the TS generated file.

## Evidence

- Wave 4 commit `b6b3631` — `test(web-console): parity test — FE can() matches Go matrix` (Plan 05 Task 5D)
  - Created `apps/web-console/tests/unit/permissions.parity.test.ts`
  - 5 actor states × 11 permissions = 55 parity assertions + 2 size guards = 57 tests total
  - Pattern B (hardcoded fixture) adopted — CI gen-permissions drift check is the runtime guard

## Notes / Deviations

- Pattern B (hardcoded fixture) was adopted instead of Pattern A (extended emitter
  that writes the parity fixture). The VALIDATION.md noted Pattern A as the preferred
  approach, but Pattern B is simpler and sufficient given that the CI codegen-drift
  check (`make gen-permissions && git diff --exit-code`) catches any registry change
  that would invalidate the fixture.
