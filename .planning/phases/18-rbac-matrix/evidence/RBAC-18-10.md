# RBAC-18-10 — Playwright rbac-unverified.spec.ts covering /billing + /api-keys unverified paths

## Scope

`apps/web-console/tests/e2e/rbac-unverified.spec.ts` covers two unverified-user
access-control scenarios: (1) visiting `/console/members` redirects to profile
settings; (2) visiting `/console/api-keys` hides the create-key form (the
`can(viewer, 'api_keys.write')` gate). The spec mirrors the `auth-shell.spec.ts`
fixture reset and `signIn` pattern, and includes a `test.skip` guard on missing
`E2E_UNVERIFIED_EMAIL` / `E2E_UNVERIFIED_PASSWORD` env vars. TypeScript compilation
(`tsc --noEmit`) is clean. Browser runtime executes in CI's `web-e2e` job.

## Notes / Deviations

- Playwright spec typechecks cleanly in the dev environment but browser execution
  requires a live stack with real unverified-user credentials (`E2E_UNVERIFIED_EMAIL`
  / `E2E_UNVERIFIED_PASSWORD`). This is an accepted operational caveat — the spec
  exits via `test.skip` when env vars are absent, and the `web-e2e` CI job provides
  the live environment. This is not a phase gap.
- The spec covers `/members` (redirect) and `/api-keys` (form hidden). It does not
  cover `/billing` directly because `/billing` is the budgets page, which now
  intentionally allows unverified owners to view (`billing.view` RequiresVerified=false).
  The redirect behaviour is on member-restricted routes only.
