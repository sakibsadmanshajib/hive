# RBAC-18-06 — web-console can(viewer, perm) helper + consumer migration (sidebar / 3 pages)

## Scope

`apps/web-console/lib/viewer-gates.ts` was rewritten to export a single `can(viewer, perm)`
helper backed by a `Set<string>` lookup against `viewer.permissions`. The legacy
boolean exports `canInviteMembers`, `canManageApiKeys`, and `allowedUnverifiedRoutes`,
plus the `ViewerGates` interface, were deleted. Three page consumers (`api-keys/page.tsx`,
`api-keys/[id]/limits/page.tsx`, `members/page.tsx`) were migrated to use `can()`.
The `client.ts` decoder was updated to decode `viewer.permissions` as `string[]`
instead of the old `gates` object. `types.ts` re-exports `Permission` from codegen.
The vitest suite was rewritten to a 55-cell matrix (5 actors × 11 permissions).
