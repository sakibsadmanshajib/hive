# RBAC-18-03 — backend handler migration (8 modules through policy.Can / RequirePermission middleware)

## Scope

All 8 control-plane handler packages that previously used bare `EmailVerified`,
`chosen.Role == "owner"`, or Gates booleans were migrated to route authz through
`policy.Can(actor, perm)`. The 8 modules are: `accounts`, `apikeys`, `accounting`,
`budgets`, `ledger`, `profiles`, `usage`, and the server entry-point (`cmd/server/main.go`
for the `RequirePlatformAdmin` → `RequirePermission(PermPlatformAdmin)` swap). An
`ActorFor()` adapter builds an `authz.Actor` from the viewer context inline in each
handler, then delegates to the stateless policy.

## Evidence

- Wave 2 commit `39adcf3` — `feat(accounts): add ActorFor adapter from viewer/membership to authz.Actor`
- Wave 2 commit `40ad4ff` — `feat(server): wire RequirePermission(PermPlatformAdmin) replacing RequirePlatformAdmin`
- Wave 3 commit `440e831` — `refactor(accounts,apikeys): route authz through policy.Can` (Plan 03 Task 3A)
- Wave 3 commit `ac8a3e3` — `refactor(18-03): migrate remaining 6 modules through policy.Can` (Plan 03 Task 3B)
