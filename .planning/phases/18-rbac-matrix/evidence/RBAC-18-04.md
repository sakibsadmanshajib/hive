# RBAC-18-04 — viewer-response wire flip: permissions:[]string in, gates.* and allowedUnverifiedRoutes out

## Scope

The `/viewer` JSON response shape was hard-flipped in a single commit: the `Gates`
struct (carrying `CanInviteMembers`, `CanManageAPIKeys` booleans) was deleted from
`types.go`, `service.go`, and `http.go`. `ViewerContext.Permissions []string` replaced
it, populated via `policy.AllGranted(actor)` — a sorted, nil-safe string slice of
every permission the actor holds. A new regression-guard test recursively walks the
full JSON tree to assert that `gates`, `can_invite_members`, `can_manage_api_keys`,
and `allowedUnverifiedRoutes` never reappear at any nesting level.

## Evidence

- Wave 3 commit `6eec4df` — `feat(accounts): replace viewer.gates.* with permissions: []` (Plan 04 Task 4A)
  - `types.go`: Gates struct deleted; `Permissions []string` added to ViewerContext
  - `service.go`: gates derivation block removed; `AllGranted(actor)` called
  - `http.go`: `"gates":map` replaced with `"permissions":vc.Permissions`
- Wave 3 commit `ed0aa93` — `test(accounts): wire-shape regression guard — no gates, permissions present` (Plan 04 Task 4B)
  - `service_wire_shape_test.go` created: `TestViewerResponseWireShape_NoGates_NoLegacy` (recursive JSON walk)
  - `TestViewerResponseWireShape_PermissionsIsArrayOfPermStrings` (validates each string is a known Permission const)
