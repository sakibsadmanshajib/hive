# PAY-14-12 — math/big enforced on every BDT subunit arithmetic path

## Truth

Every monetary computation in Phase 14 — budget cap comparison, MTD
aggregation, threshold crossing math, invoice totals, grant amounts —
uses `math/big.Int`. Float64 is forbidden in these packages; a grep
audit returns empty.

## Implementation

- BDT subunits stored as `BIGINT` in Postgres; transported as decimal
  strings on the wire; parsed into `*big.Int` in Go.
- `budgets.ThresholdCrossed(softCap, mtdSpend *big.Int, pct int) bool`
  uses `big.Int.Mul` + `big.Int.Cmp` only.
- `limits.BudgetGate.Check` uses `big.Int.Cmp(hardCap, mtdSpend) <= 0`.
- `grants.Service.Create` validates `amount.Sign() > 0` against
  `*big.Int`.
- `invoices.Service.Aggregate` sums per-entry deltas via
  `big.Int.Add` over the credit ledger window.
