# PAY-14-08 — Phase-18 RBAC contract stub: platform.IsWorkspaceOwner / IsPlatformAdmin single source-of-truth

## Truth

Every Phase 14 owner-/admin-gated handler imports
`apps/control-plane/internal/platform` and calls
`RoleService.IsWorkspaceOwner` / `IsPlatformAdmin` exactly once. The function
SIGNATURES are the Phase 18 contract — bodies may be replaced with the
tier-aware permission matrix evaluator without changing call sites.

## Implementation

Task 2 seeded the `platform.RoleService` (interface + signatures + middleware).
Task 5 closes the loop by:

1. Seeding the concrete pgxpool-backed `RoleStore` at
   `apps/control-plane/internal/platform/role_pgx.go::NewPgxRoleStore`. Queries
   `public.account_memberships` for membership role and joins
   `public.accounts.is_platform_admin` for the platform-admin flag.
2. Wiring `roleSvc = platform.NewRoleService(NewPgxRoleStore(pool))` in
   `apps/control-plane/cmd/server/main.go`.
3. Using `roleSvc.RequirePlatformAdmin` middleware on the admin grant routes.
4. Using `roleSvc` (which satisfies `grants.AdminChecker`) as the service-layer
   admin checker so every grant create call funnels through one predicate.

## Single-call discipline

- `grants.Service.Create` calls `s.admin.IsPlatformAdmin(ctx, in.GrantedByUserID)`
  exactly once per request. No redundant predicate calls.
- `platform.RequirePlatformAdmin` middleware also calls IsPlatformAdmin
  exactly once per request — this is the gate, separate from any defensive
  service-layer recheck.
- Both call sites resolve to the same `RoleStore` so Phase 18 swap is
  trivial.

## Tests

`platform/role_test.go` covers all behaviour cases:
- Owner role → true
- Member / unknown role → false
- Stranger (no membership) → false
- Missing workspace → ErrWorkspaceNotFound
- Store error → propagated
- Platform-admin flag set → true
- Default (unset) → false
- Middleware: admin passes, non-admin 403, unauth 401, store-error 500
