# PAY-14-10 — Invoice PDF download zero USD/FX strings

## Truth

The rendered invoice PDF MUST be 100% BDT-only on every customer-visible
surface — header, line items, totals, footer. No `$`, no `USD`, no
`fx_*`, no `exchange_rate` strings. Regulatory rule: never expose FX or
USD language to BD customers.

## Implementation

- `invoices.GofpdfRenderer` writes only BDT subunits formatted with
  `formatBDT(*big.Int)` ("৳1,23,45,678.90"). No locale-derived currency
  symbols leak.
- `invoices.PDFRenderer.scrubInputs` (`pdf.go:185-200`) defines a
  `BANNED_TOKENS` array (`amount_usd`, `usd_*`, `fx_*`, `price_per_credit_usd`,
  `exchange_rate`, `$`, `USD`) and rejects any input string containing
  any token before passing it to gofpdf. Tripwire on Phase 17
  customer-USD lint.
- `pdf_test.go::TestRender_RoundTripPDFTextStripsBannedTokens` extracts
  text from a generated PDF using `github.com/ledongthuc/pdf` and asserts
  zero matches for the banned set.

## Tests

- `TestPDFRenderer_RejectsBannedInputTokens`
- `TestRender_RoundTripPDFTextStripsBannedTokens`
- `TestService_Generate_PDFExtractIsBDTOnly` — full round-trip from
  ledger aggregation to extracted PDF text.
