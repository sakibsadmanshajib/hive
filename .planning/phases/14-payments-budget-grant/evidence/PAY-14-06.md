# PAY-14-06 — Non-owner POST /v1/admin/credit-grants → 403 provider-blind

## Truth

Non-platform-admin POST attempts to `/v1/admin/credit-grants` MUST receive a
provider-blind sanitised 403 response. No leak of "must be admin" / "you do not
own this workspace" — opaque insufficient-permissions message only.

## Implementation

- `platform.RequirePlatformAdmin` middleware wraps the admin route group in
  main.go. Non-admins receive `{"error":"platform admin permission required"}`
  with HTTP 403 (provider-blind).
- Defensive layer: even if middleware is bypassed, `grants.Service.Create`
  returns `ErrForbidden`, which the handler maps to HTTP 403 with the
  sanitised body `{"error":"insufficient permissions"}`.
- Unauthenticated requests return HTTP 401 with `{"error":"authentication required"}`.

## Tests

- `TestRequirePlatformAdmin_NonAdminGets403` (platform package)
- `TestHandlerCreate_NonAdminForbidden` (grants package, via gate)
- `TestHandlerCreate_Unauthenticated` (grants package, no viewer)
- `TestHandlerCreate_ServiceForbiddenWhenGateBypassed` (defensive: service
  layer surfaces 403 even if middleware bypassed in test harness)

## Provider-blind

The 403 body is fixed sanitised JSON; no provider names, stack traces, or
"you must be owner" strings leak. Verified by visual inspection of
`role.go::writeForbidden` and `http.go::handleCreate` error map.
