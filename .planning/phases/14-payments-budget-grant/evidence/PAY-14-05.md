# PAY-14-05 — Owner-only POST /v1/admin/credit-grants resolves grantee + immutable audit row + same-tx ledger append

## Truth

Workspace owner (platform admin) issues discretionary credit grants. The grant
row is the audit log itself (no separate audit pipeline because schema-level
immutability gives the durability guarantee). One grant produces exactly one
ledger entry; both writes share a single Postgres transaction.

## Implementation

- `grants.Service.Create` enforces owner gate via narrow `AdminChecker` port
  (mirrors `platform.RoleService.IsPlatformAdmin`).
- `grants.Repository.CreateWithLedger` runs a single `pgx.Tx` covering:
  1. `INSERT INTO public.credit_idempotency_keys (operation_type='grant', ...)`
  2. `INSERT INTO public.credit_ledger_entries (entry_type='grant', credits_delta=+amount, ...)`
  3. `UPDATE public.credit_idempotency_keys SET ledger_entry_id = $3 WHERE ...`
  4. `INSERT INTO public.credit_grants (granted_by_user_id, granted_to_user_id, granted_to_workspace_id, amount_bdt_subunits, reason_note, ledger_entry_id, currency)`
  Failure at any step rolls the entire transaction back — no orphaned grant
  rows, no orphaned ledger entries.
- BDT-only wire shape (`amount_bdt_subunits` as decimal string for math/big
  invariant). Zero `amount_usd|usd_|fx_|exchange_rate|price_per_credit_usd`
  keys (asserted by `TestHandlerCreate_AdminSuccess`).

## Tests

- `TestCreate_AdminHappyPath` — owner POST creates grant + ledger entry
- `TestCreate_LedgerErrorRollsBack` — single-tx rollback on ledger failure
- `TestHandlerCreate_AdminSuccess` — HTTP 201 + provider-blind body
