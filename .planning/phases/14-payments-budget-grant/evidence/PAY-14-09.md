# PAY-14-09 — Monthly invoice cron generates BDT-only PDF per workspace

## Truth

A monthly cron fires at the start of each calendar month, walks every
workspace with usage activity in the prior period, and produces a single
BDT-only invoice row + PDF artifact stored in Supabase Storage
(`hive-files` bucket). Idempotent per `(workspace_id, period_start)` via
unique constraint.

## Implementation

- Schema: `public.invoices` (`workspace_id`, `period_start`, `period_end`,
  `total_bdt_subunits BIGINT`, `currency='BDT'`, `pdf_storage_key`,
  `generated_at`) with `UNIQUE (workspace_id, period_start)` enforcing
  cron idempotency.
- `invoices.Service.Generate` aggregates `credit_ledger_entries` for the
  period via `*big.Int` sum (no float64), renders the PDF through
  `invoices.GofpdfRenderer`, uploads via `invoices.StorageAdapter` to the
  `hive-files` bucket, and writes the row.
- `invoices.Cron` ticks hourly; on the first run after the period boundary
  it sweeps every workspace with usage in `[period_start, period_end)` and
  calls `Service.Generate`. Per-workspace error isolation: one workspace
  failure does not block the rest of the sweep.
