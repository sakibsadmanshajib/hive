**A. Create `apps/edge-api/internal/inference/orchestrator.go`.**

Define `Orchestrator` struct with fields:
- `authorizer` *authz.Authorizer
- `routing` *RoutingClient
- `accounting` *AccountingClient
- `litellm` *LiteLLMClient

Constructor: `NewOrchestrator(authorizer *authz.Authorizer, routing *RoutingClient, accounting *AccountingClient, litellm *LiteLLMClient) *Orchestrator`.

Method `executeSync(ctx context.Context, w http.ResponseWriter, r *http.Request, endpoint EndpointType, body []byte, model string, needFlags NeedFlags, estimatedCredits int64, dispatch func(ctx context.Context, litellmModel string, body []byte) (*http.Response, error), normalize func(respBody []byte, aliasID string) ([]byte, *UsageResponse, error)) error`:

The lifecycle:
1. **Authorize** -- call `o.authorizer.Authorize(ctx, authHeader, model, estimatedCredits, 0, 0)` directly. Handle the error by writing the OpenAI error and returning. Follow the exact same error-mapping logic as the existing `authorizeAliasRequest` in main.go (invalid_api_key -> 401, insufficient_quota -> 429, model_not_found -> 404, rate_limit_exceeded -> 429 with headers).
2. **Select route** -- call `o.routing.SelectRoute(ctx, ...)` with `AliasID: model` and the needFlags. On `ErrAliasNotFound`, call `writeModelNotFoundError`. On `ErrRouteNotEligible`, return 400 with capability mismatch message.
3. **Start attempt** -- call `o.accounting.StartAttempt(ctx, ...)` with `AccountID` from snapshot, `RequestID` (generate UUID), `AttemptNumber: 1`, `Endpoint: endpoint`, `ModelAlias: model`, `Status: "dispatching"`, `APIKeyID` from snapshot.KeyID.
4. **Create reservation** -- call `o.accounting.CreateReservation(ctx, ...)` with estimated credits. On policy error, write 429 insufficient_quota error and return.
5. **Dispatch** -- call the `dispatch` function with `route.LiteLLMModelName` and `body`. On error or non-2xx: call `o.accounting.ReleaseReservation(...)` with reason `"upstream_error"`, record error usage event, write provider-blind upstream error.
6. **Read response body** -- read LiteLLM response body fully (with 10MB limit via `io.LimitReader`).
7. **Normalize** -- call the `normalize` function to strip provider-specific fields and extract usage. On normalize error, release reservation and write 502.
8. **Finalize** -- call `o.accounting.FinalizeReservation(...)` with actual credits computed from usage tokens. Record completed usage event with token counts.
9. **Write response** -- write normalized JSON with `Content-Type: application/json` and 200 status.

Use `defer` to ensure reservation cleanup: if the function returns without finalizing, release the reservation with reason `"interrupted"`.

**B. Create `apps/edge-api/internal/inference/chat_completions.go`.**

Function `handleChatCompletions(o *Orchestrator, w http.ResponseWriter, r *http.Request)`:
1. Read request body with 10MB limit.
2. Parse into `ChatCompletionRequest` (only decode the fields needed: Model, Stream, StreamOptions, ReasoningEffort, Tools presence, ResponseFormat presence).
3. Validate: `Model` must not be empty (call `writeMissingFieldError`). `Messages` must be present.
4. If `Stream` is true, return a proper OpenAI-style error: `{"error":{"message":"Streaming is not yet available.","type":"api_error","param":null,"code":"not_implemented"}}`.
5. Build `NeedFlags`: `NeedChatCompletions: true`, `NeedStreaming: req.Stream`, `NeedReasoning: req.ReasoningEffort != nil`.
6. Estimate credits: use a fixed estimate of 10000 credits (1 cent equivalent) for reservation. Actual usage computed from response.
7. Call `o.executeSync(...)` with `dispatch: o.litellm.ChatCompletion` and normalize function.

The normalize function for chat/completions:
- Unmarshal LiteLLM response into `ChatCompletionResponse`.
- Overwrite `Model` field with the Hive alias ID (not the LiteLLM route handle).
- Ensure `Object` is `"chat.completion"`.
- Strip any unknown top-level fields by marshaling only the known struct fields (allowlist, not blocklist).
- Return the cleaned JSON and extracted `UsageResponse`.

**C. Create `apps/edge-api/internal/inference/completions.go`.**

Function `handleCompletions(o *Orchestrator, w http.ResponseWriter, r *http.Request)`:
1. Read and parse `CompletionRequest`. Validate `Model` and `Prompt` are present.
2. If `Stream` is true, return not-implemented error (same as above, temporary).
3. Build `NeedFlags`: `NeedChatCompletions: true` (LiteLLM routes legacy completions through chat/completions-capable routes).
4. Call `o.executeSync(...)` with `dispatch: o.litellm.Completion`.

Normalize function for completions:
- Unmarshal into `CompletionResponse`.
- Overwrite `Model` with Hive alias.
- Ensure `Object` is `"text_completion"`.
- Allowlist known fields only.
