**A. Create `apps/edge-api/internal/inference/orchestrator.go`.**

Define `Orchestrator` struct with fields:
- `authorizer` *authz.Authorizer
- `routing` *RoutingClient
- `accounting` *AccountingClient
- `litellm` *LiteLLMClient

Constructor: `NewOrchestrator(authorizer *authz.Authorizer, routing *RoutingClient, accounting *AccountingClient, litellm *LiteLLMClient) *Orchestrator`.

Method `executeSync(ctx context.Context, w http.ResponseWriter, r *http.Request, endpoint EndpointType, body []byte, model string, needFlags NeedFlags, estimatedCredits int64, dispatch func(ctx context.Context, litellmModel string, body []byte) (*http.Response, error), normalize func(respBody []byte, aliasID string) ([]byte, *UsageResponse, error)) error`:

The lifecycle:
1. **Authorize** -- call the package-level `authorizeAliasRequest` helper (import from main package is not possible, so extract the helper into a shared function or accept the authorizer and call `a.authorizer.Authorize(ctx, r.Header.Get("Authorization"), model, estimatedCredits, 0, 0)` directly. Handle the error by writing the OpenAI error and returning.
2. **Select route** -- call `o.routing.SelectRoute(ctx, ...)` with `AliasID: model` and the needFlags. On `ErrAliasNotFound`, call `writeModelNotFoundError`. On `ErrRouteNotEligible`, return 400 with capability mismatch message.
3. **Start attempt** -- call `o.accounting.StartAttempt(ctx, ...)` with `AccountID` from snapshot, `RequestID` (generate UUID), `AttemptNumber: 1`, `Endpoint: endpoint`, `ModelAlias: model`, `Status: "dispatching"`, `APIKeyID` from snapshot.KeyID.
4. **Create reservation** -- call `o.accounting.CreateReservation(ctx, ...)` with estimated credits. On policy error, write 429 insufficient_quota error and return.
5. **Dispatch** -- call the `dispatch` function with `route.LiteLLMModelName` and `body`. On error or non-2xx: call `o.accounting.ReleaseReservation(...)` with reason `"upstream_error"`, record error usage event, write provider-blind upstream error.
6. **Read response body** -- read LiteLLM response body fully (with 10MB limit via `io.LimitReader`).
7. **Normalize** -- call the `normalize` function to strip provider-specific fields and extract usage. On normalize error, release reservation and write 502.
8. **Finalize** -- call `o.accounting.FinalizeReservation(...)` with actual credits computed from usage tokens. Record completed usage event with token counts.
9. **Write response** -- write normalized JSON with `Content-Type: application/json` and 200 status.
