**A. Add internal (non-auth-gated) accounting endpoints to control-plane.**

The edge-api authenticates via API key, not Supabase auth. It cannot call the auth-gated `/api/v1/accounts/current/credits/reservations/*` endpoints. Add internal service-to-service endpoints following the same pattern as `/internal/routing/select` and `/internal/apikeys/resolve`.

In `apps/control-plane/internal/accounting/http.go`, add new handler methods:
- `handleInternalCreateReservation` at path `POST /internal/accounting/reservations` -- accepts `account_id` in the JSON body directly (not from Supabase viewer context). JSON body fields: `account_id` (string UUID), `request_id` (string), `attempt_number` (int), `api_key_id` (string, optional), `endpoint` (string), `model_alias` (string), `estimated_credits` (int64), `policy_mode` (string). Returns the Reservation JSON.
- `handleInternalFinalizeReservation` at path `POST /internal/accounting/reservations/finalize` -- accepts `account_id`, `reservation_id`, `actual_credits`, `terminal_usage_confirmed`, `status` in JSON body. Returns the Reservation JSON.
- `handleInternalReleaseReservation` at path `POST /internal/accounting/reservations/release` -- accepts `account_id`, `reservation_id`, `reason` in JSON body. Returns the Reservation JSON.

These handlers call the same `h.svc.CreateReservation(...)`, `h.svc.FinalizeReservation(...)`, `h.svc.ReleaseReservation(...)` methods -- the only difference is they parse `account_id` from the JSON body instead of resolving from Supabase viewer context.

In the `ServeHTTP` switch, add three new cases for the `/internal/accounting/reservations*` paths.

**B. Add internal usage endpoints.**

In `apps/control-plane/internal/usage/http.go`, add:
- `handleInternalStartAttempt` at path `POST /internal/usage/attempts` -- accepts `account_id`, `request_id`, `attempt_number`, `endpoint`, `model_alias`, `status`, `api_key_id` (optional) in JSON body. Calls `h.svc.StartAttempt(...)`. Returns the RequestAttempt JSON.
- `handleInternalRecordEvent` at path `POST /internal/usage/events` -- accepts all `RecordEventInput` fields plus `account_id` in JSON body. Calls `h.svc.RecordEvent(...)`. Returns the UsageEvent JSON.

In the usage `ServeHTTP` switch, add two new cases for `/internal/usage/*` paths.

**C. Register internal routes in the platform router.**

In `apps/control-plane/internal/platform/http/router.go`, register the new internal routes WITHOUT auth middleware (same pattern as `/internal/routing/select` and `/internal/apikeys/resolve`):
```go
if cfg.AccountingHandler != nil {
    mux.Handle("/internal/accounting/reservations", cfg.AccountingHandler)
    mux.Handle("/internal/accounting/reservations/finalize", cfg.AccountingHandler)
    mux.Handle("/internal/accounting/reservations/release", cfg.AccountingHandler)
}
if cfg.UsageHandler != nil {
    mux.Handle("/internal/usage/attempts", cfg.UsageHandler)
    mux.Handle("/internal/usage/events", cfg.UsageHandler)
}
```
Place these BEFORE the auth-gated registrations so `/internal/*` routes are not accidentally wrapped.
