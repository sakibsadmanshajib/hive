# CAP-16-01 Evidence — `ensureCapabilityColumns` no longer targets the wrong table

## Behavior

`ensureCapabilityColumns` is not defined or called in
`apps/control-plane/internal/routing/repository.go`. The provider capability
schema lives in Supabase migrations
(`supabase/migrations/20260414_01_provider_capabilities_media_columns.sql`),
which correctly targets `public.provider_capabilities` (NOT
`route_capabilities`). `ListRouteCandidates` joins
`public.provider_capabilities` via `route_id` and selects every capability
column the routing service needs — including the five media + batch columns
added by Phase 14. A regression-guard test
(`TestRoutingRepositoryDoesNotRunCapabilityDDL`) asserts the function and any
runtime `ALTER TABLE` statement cannot reappear in `repository.go`.

The original latent bug recorded in `CLAUDE.md` Known Issues §1 — that
`ensureCapabilityColumns` issued DDL against `route_capabilities` instead of
`provider_capabilities` — was eliminated by Phase 14's media-columns work
(2026-04-14): the DDL was lifted out of the runtime path into a Supabase
migration, and `repository_schema_test.go` was added to prevent the function
from returning. Phase 16 verifies that closure.

## Code paths

- `apps/control-plane/internal/routing/repository.go` — routing repository.
  `ListRouteCandidates` selects 13 capability columns (responses, chat
  completions, completions, embeddings, streaming, reasoning, cache_read,
  cache_write, image_generation, image_edit, tts, stt, batch) via
  `JOIN public.provider_capabilities c ON c.route_id = r.route_id`. No
  `ensureCapabilityColumns` function. No `ALTER TABLE` at runtime.
- `apps/control-plane/internal/routing/repository_schema_test.go` — guard
  tests:
  - `TestRoutingRepositoryDoesNotRunCapabilityDDL` — asserts neither
    `ensureCapabilityColumns` nor `ALTER TABLE` appears in `repository.go`.
  - `TestProviderCapabilitiesMigrationAddsMediaColumns` — asserts the
    migration alters `public.provider_capabilities` (and rejects any
    migration altering `route_capabilities`).
  - `TestProviderCapabilitiesMigrationBackfillsMediaAndBatchRoutes` — asserts
    the migration backfills `route-openrouter-auto` with all five media +
    batch columns set true.
  - `TestListRouteCandidatesSelectsMediaColumns` — asserts the SELECT clause
    pulls `c.supports_image_generation`, `c.supports_image_edit`,
    `c.supports_tts`, `c.supports_stt`, `c.supports_batch`.
- `supabase/migrations/20260414_01_provider_capabilities_media_columns.sql` —
  `alter table public.provider_capabilities add column if not exists ...` for
  the five new columns and a backfill `update ... set ... = true where
  route_id = 'route-openrouter-auto'`. Targets the correct table.
