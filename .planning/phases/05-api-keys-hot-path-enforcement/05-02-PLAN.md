<objective>
Add per-key policy controls for expiration, allowed models, and budgets, then project those controls into a Redis-backed auth snapshot the edge can enforce cheaply.

Purpose: Phase 5 is not complete when keys merely exist. Each key needs explicit policy state that resolves into Hive aliases, budget guardrails, and a hot-path authorization snapshot the edge can consume without synchronous database reads.
Output: durable policy tables, snapshot hydration in the control plane, and an edge `authz` package that validates API keys against Redis-backed policy data.
</objective>
