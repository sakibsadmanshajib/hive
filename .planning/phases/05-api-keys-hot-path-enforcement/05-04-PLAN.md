# Phase 05 Gap Closure Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the Phase 05 UAT gaps around stale edge authorization after key mutation and missing end-to-end API-key usage attribution.

**Architecture:** Keep the fix inside the existing Phase 05 boundaries. The control-plane should own auth-snapshot invalidation because it already owns the source of truth for key state, while the accounting and usage surfaces need minimal schema-preserving HTTP and response changes so `api_key_id` can travel through the existing reservation/finalize path.

**Tech Stack:** Go, net/http, pgx, go-redis, Docker Compose

---

<objective>
Close the two diagnosed Phase 05 UAT gaps without expanding scope beyond API-key lifecycle, hot-path auth, and accounting attribution.

Purpose: Phase 05 cannot be considered done while revoked secrets stay authorized or while the public accounting flow cannot attribute usage to a key.
Output: one focused hardening slice that makes auth snapshots immediately consistent after key mutations and makes `api_key_id` plus `last_used_at` observable through the current accounting and usage APIs.
</objective>
