5. Implement secret handling in `service.go` exactly like this:
   - Generate `32` random bytes with `crypto/rand`.
   - Encode them with `base64.RawURLEncoding`.
   - Prefix the result with `hk_`.
   - Compute `token_hash` as the lowercase hex SHA-256 of the full `hk_...` secret.
   - Store only `token_hash` plus `redacted_suffix`, where `redacted_suffix` is the last `6` characters of the raw secret.
   - `CreateKey` and `RotateKey` return the raw secret once in memory and never log it, persist it, or include it in list responses.

6. Implement lifecycle rules exactly:
   - if `expires_at` is in the past when reading a key whose stored `status` is `active`, expose `KeyStatusExpired` from the service without mutating sibling keys
   - `DisableKey` sets `status = 'disabled'` and `disabled_at = now()`
   - `EnableKey` clears `disabled_at` and restores `status = 'active'`
   - `RevokeKey` sets `status = 'revoked'`, `revoked_at = now()`, and cannot be undone
   - `RotateKey` creates a brand-new key row, returns a brand-new raw secret, and immediately revokes only the rotated source key
