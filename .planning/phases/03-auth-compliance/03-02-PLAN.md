Create `apps/api/test/helpers/test-app.ts` — a helper that boots a real Fastify server with mock services for integration testing. The helper must:

1. Create a Fastify instance the same way `createApp` does (same AJV options, TypeBox type provider) but WITHOUT cors and WITHOUT `createRuntimeServices()`.
2. Accept mock services as a parameter so tests can control `resolveApiKey` behavior.
3. Register routes via `registerRoutes(app, mockServices)` so the full v1Plugin (error handler, onSend hook, routes) is active.
4. Listen on port 0 (ephemeral) and return the address.
5. Provide a cleanup function to close the server.

```typescript
import Fastify from "fastify";
import { TypeBoxTypeProvider } from "@fastify/type-provider-typebox";
import { registerRoutes } from "../../src/routes";
import type { RuntimeServices } from "../../src/runtime/services";

export type MockServices = {
  users: {
    resolveApiKey: (key: string) => Promise<{ userId: string; scopes: string[]; apiKeyId?: string } | null>;
  };
  env: {
    allowDevApiKeyPrefix: boolean;
    supabase: { flags: { authEnabled: boolean } };
  };
  supabaseAuth: {
    getSessionPrincipal: () => Promise<null>;
  };
  authz: {
    requirePermission: () => Promise<boolean>;
  };
  userSettings: {
    getForUser: () => Promise<{ apiEnabled: boolean }>;
    canUse: () => boolean;
  };
};

export function createMockServices(validApiKey: string, userId: string): MockServices {
  return {
    users: {
      resolveApiKey: async (key: string) => {
        if (key === validApiKey) {
          return { userId, scopes: ["chat", "image", "usage", "billing"], apiKeyId: "key_test" };
        }
        return null;
      },
    },
    env: {
      allowDevApiKeyPrefix: false,
      supabase: { flags: { authEnabled: false } },
    },
    supabaseAuth: {
      getSessionPrincipal: async () => null,
    },
    authz: {
      requirePermission: async () => true,
    },
    userSettings: {
      getForUser: async () => ({ apiEnabled: true }),
      canUse: () => true,
    },
  };
}

export async function createTestApp(mockServices: MockServices) {
  const app = Fastify({
    logger: false,
    ajv: {
      customOptions: {
        removeAdditional: false,
      },
    },
  }).withTypeProvider<TypeBoxTypeProvider>();

  registerRoutes(app, mockServices as unknown as RuntimeServices);

  const address = await app.listen({ port: 0 });
  return { app, address };
}
```

Key design decisions:
- `logger: false` to avoid noise in test output
- Mock services use the exact same shape as RuntimeServices but only implement what auth needs
- `resolveApiKey` is the key mock — returns a match for the known valid key, null otherwise
- `supabase.flags.authEnabled: false` ensures the JWT path in old `resolvePrincipal` is skipped (only matters for non-v1 routes tested incidentally)
- Cast `mockServices as unknown as RuntimeServices` because we only mock what's needed
- The `createMockServices` factory takes the valid key and userId so tests can set up specific scenarios

Create `apps/api/test/routes/v1-auth-compliance.test.ts` — SDK integration tests using the real `openai` npm package as the HTTP client.

The test file must:

1. Use `beforeAll` to boot the test app with `createTestApp(createMockServices("sk-test-valid", "user_test"))`.
2. Use `afterAll` to call `app.close()`.
3. Create an `OpenAI` client instance pointing at the test server's address.

Test cases (each as a separate `it` block):

**Auth tests (FOUND-02):**

a) `"valid bearer token authenticates successfully"`:
   - Create client with `apiKey: "sk-test-valid"`, `baseURL: address + "/v1"`
   - Call `client.models.list()` (public endpoint, will succeed regardless of auth)
   - Expect response to have `data` array (proves server is reachable and SDK works)

b) `"missing Authorization header returns 401 with correct error body"`:
   - Use raw `fetch` (not SDK) to `GET ${address}/v1/chat/completions` with no Authorization header, method POST, body `{"model":"test","messages":[]}`
   - OR use SDK with empty apiKey — but the SDK requires a non-empty apiKey
   - Best approach: use `fetch` directly for this case
   - Assert response status is 401
   - Assert response body is `{ error: { message: "No API key provided", type: "authentication_error", param: null, code: "invalid_api_key" } }`

c) `"invalid bearer token returns 401 with correct error body"`:
   - Create client with `apiKey: "sk-wrong-key"`, `baseURL: address + "/v1"`
   - Call `client.chat.completions.create({ model: "test", messages: [] })` inside try/catch
   - Assert the caught error is `instanceof OpenAI.AuthenticationError`
   - Assert `error.status === 401`
   - Assert `error.error` contains `{ message: "Incorrect API key provided", type: "authentication_error", code: "invalid_api_key" }`

d) `"x-api-key header is ignored when Bearer is present"`:
   - Use `fetch` directly: `POST ${address}/v1/chat/completions` with both `Authorization: Bearer sk-test-valid` AND `x-api-key: sk-wrong-key` headers, body `{"model":"test","messages":[]}`
   - Assert response status is NOT 401 (the Bearer token is valid, x-api-key should be ignored)
   - This proves the v1 auth path only looks at Bearer, not x-api-key

e) `"SDK throws AuthenticationError not generic APIError for 401"`:
   - Create client with `apiKey: "sk-bad"`, `baseURL: address + "/v1"`
   - Call any endpoint, catch the error
   - Assert `error instanceof OpenAI.AuthenticationError` is `true`
   - Assert `error instanceof OpenAI.APIError` is also `true` (AuthenticationError extends APIError)

**Content-Type test (FOUND-05):**

f) `"non-streaming response has Content-Type: application/json"`:
   - Use `fetch` directly: `GET ${address}/v1/models` (public endpoint, no auth needed)
   - Assert `response.headers.get("content-type")` includes `application/json`

**Important implementation notes:**
- The `models.list()` endpoint works without auth (models route has no auth check)
- For endpoints that require auth and will fail downstream (e.g., chat completions with valid auth but no real provider), the response may be a 500 — that's fine, the test only checks that auth PASSED (status is not 401)
- Use `fetch` for cases where the SDK can't easily reproduce the scenario (missing header, dual headers)
- Import `OpenAI` from `"openai"` — installed as devDependency in plan 01
- Import `createTestApp, createMockServices` from `"../helpers/test-app"`
