Create `apps/api/src/routes/api-error.ts`:
- Export type `OpenAIErrorType` as a union of: `"invalid_request_error" | "authentication_error" | "permission_error" | "not_found_error" | "rate_limit_error" | "insufficient_quota" | "server_error"`
- Export const `STATUS_TO_TYPE: Record<number, OpenAIErrorType>` mapping: 400="invalid_request_error", 401="authentication_error", 402="insufficient_quota", 403="permission_error", 404="not_found_error", 429="rate_limit_error"
- Export type `ApiErrorOpts = { type?: OpenAIErrorType; param?: string | null; code?: string | null }`
- Export function `sendApiError(reply: FastifyReply, status: number, message: string, opts?: ApiErrorOpts): void` that calls `reply.code(status).send({ error: { message, type: opts?.type ?? STATUS_TO_TYPE[status] ?? "server_error", param: opts?.param ?? null, code: opts?.code ?? null } })`

Create `apps/api/src/routes/v1-plugin.ts`:
- Import `FastifyInstance`, `RuntimeServices`, `sendApiError`, `STATUS_TO_TYPE` from api-error
- Export async function `v1Plugin(app: FastifyInstance, opts: { services: RuntimeServices })` as a Fastify plugin
- Inside the plugin, call `app.setErrorHandler((error, request, reply) => { ... })` that:
  - Gets status from `error.statusCode ?? 500`
  - Gets message from `error.message || "Internal server error"`
  - Calls `reply.code(status).send({ error: { message, type: STATUS_TO_TYPE[status] ?? "server_error", param: null, code: null } })`
- Inside the plugin, call `app.setNotFoundHandler((request, reply) => { ... })` that:
  - Sends 404 with message `"Unknown API route: ${request.method} ${request.url}"`, type `"not_found_error"`, param null, code null
- The plugin does NOT register any routes itself (route registration moves here in Plan 02)
- Do NOT use a `prefix` option on this plugin -- routes already contain `/v1/` in their paths

Create `apps/api/test/routes/api-error-format.test.ts`:
- Use vitest (import { describe, it, expect } from "vitest")
- Use Fastify's `fastify.inject()` pattern for testing (check existing test files for the project's test setup pattern)
- Create a test Fastify instance, register the v1Plugin, and add test routes inside it for handler-throws and domain-error scenarios
- Register a non-v1 test route outside the plugin to verify isolation
- Write tests for all behaviors listed above
