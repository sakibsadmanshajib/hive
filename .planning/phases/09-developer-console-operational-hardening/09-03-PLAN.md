**1. Add analytics client functions to `apps/web-console/lib/control-plane/client.ts`:**
NOTE: This file was already extended by Plan 02 with billing/keys/catalog functions. This step ADDS to that file — do not overwrite Plan 02's additions. Read the file first and append new functions.

New interfaces:
```typescript
export interface UsageSummaryRow {
  group_key: string;
  total_input_tokens: number;
  total_output_tokens: number;
  total_credits_spent: number;
  request_count: number;
}

export interface SpendSummaryRow {
  group_key: string;
  total_credits: number;
  entry_count: number;
}

export interface ErrorSummaryRow {
  group_key: string;
  error_count: number;
  total_requests: number;
  error_rate: number;
}

export interface BudgetThreshold {
  id: string;
  threshold_credits: number;
  alert_dismissed: boolean;
  last_notified_at: string | null;
  created_at: string;
  updated_at: string;
}
```

New functions (following existing decoder pattern — no `as` casts):
- `getAnalyticsUsage(params: { group_by: string; window?: string; from?: string; to?: string }): Promise<UsageSummaryRow[]>` — GET `/api/v1/accounts/current/analytics/usage?group_by=X&window=Y`
- `getAnalyticsSpend(params: { group_by: string; window?: string; from?: string; to?: string }): Promise<SpendSummaryRow[]>` — GET `/api/v1/accounts/current/analytics/spend?group_by=X&window=Y`
- `getAnalyticsErrors(params: { group_by: string; window?: string; from?: string; to?: string }): Promise<ErrorSummaryRow[]>` — GET `/api/v1/accounts/current/analytics/errors?group_by=X&window=Y`
- `getBudgetThreshold(): Promise<BudgetThreshold | null>` — GET `/api/v1/accounts/current/budget`, return `threshold` field (null if no threshold set)
- `upsertBudgetThreshold(thresholdCredits: number): Promise<BudgetThreshold>` — PUT `/api/v1/accounts/current/budget` with body `{"threshold_credits": N}`
- `dismissBudgetAlert(): Promise<void>` — POST `/api/v1/accounts/current/budget/dismiss`
