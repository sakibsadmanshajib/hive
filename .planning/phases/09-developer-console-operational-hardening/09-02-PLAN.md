New functions (each following the getRequestContext + fetch + decode pattern):
- `getBalance(): Promise<BalanceSummary>` — GET `/api/v1/accounts/current/credits/balance`
- `getLedgerEntries(params: { limit?: number; cursor?: string; type?: string }): Promise<LedgerPage>` — GET `/api/v1/accounts/current/credits/ledger?limit=N&cursor=X&type=Y`
- `getInvoices(): Promise<Invoice[]>` — GET `/api/v1/accounts/current/invoices`, decode `invoices` array
- `getInvoice(id: string): Promise<Invoice>` — GET `/api/v1/accounts/current/invoices/{id}`
- `getCheckoutRails(): Promise<CheckoutOptions>` — GET `/api/v1/accounts/current/checkout/rails`
- `initiateCheckout(rail: string, credits: number, idempotencyKey: string): Promise<CheckoutInitiateResponse>` — POST `/api/v1/accounts/current/checkout/initiate` with body `{rail, credits, idempotency_key}`
- `getApiKeys(): Promise<ApiKey[]>` — GET `/api/v1/accounts/current/api-keys`, decode `items` array
- `createApiKey(nickname: string, expiresAt?: string): Promise<ApiKey>` — POST `/api/v1/accounts/current/api-keys` with body `{nickname, expires_at}`
- `revokeApiKey(keyId: string): Promise<ApiKey>` — POST `/api/v1/accounts/current/api-keys/{keyId}/revoke`
- `rotateApiKey(keyId: string, nickname: string, expiresAt?: string): Promise<ApiKey>` — POST `/api/v1/accounts/current/api-keys/{keyId}/rotate` with body `{nickname, expires_at}`
- `getCatalogModels(): Promise<CatalogModel[]>` — GET `/api/v1/catalog/models`, decode `models` array

Each function MUST use the existing decoder helpers (readStringField, readArrayField, etc.) to decode responses. Build decoders analogous to `decodeViewerResponse` and `decodeAccountProfile`. Do NOT use `as` casts per project memory `feedback_strict_typescript.md`.
