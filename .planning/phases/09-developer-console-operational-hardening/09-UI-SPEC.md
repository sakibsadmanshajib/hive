# Phase 9 — UI Design Contract

> Visual and interaction contract for Phase 9: Developer Console & Operational Hardening.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Source: existing console inline-style patterns extracted from codebase (no Tailwind, no shadcn).
> All new Phase 9 pages MUST match the inline-style approach already in use.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — raw inline styles throughout |
| Preset | not applicable |
| Component library | none (no Radix, no shadcn) |
| Icon library | none (text labels only; no icon font or SVG sprite installed) |
| Font | system-ui / sans-serif (browser default inherited) |

**Rationale (source: codebase scan 2026-04-10):** No `components.json`, no `tailwind.config.*`, no `postcss.config.*` found in `apps/web-console/`. All existing console pages use `style={{}}` inline objects. Phase 9 must follow the same pattern — no new design system should be introduced mid-product.

## Spacing Scale

Derived from existing inline style values across `layout.tsx`, `nav-shell.tsx`, and settings pages.
All values are multiples of 4px.

| Token | px Value | rem Equivalent | Usage |
|-------|----------|----------------|-------|
| xs | 4px | 0.25rem | Icon-adjacent gaps, tight inline spacing (`gap: "0.25rem"`) |
| sm | 8px | 0.5rem | Nav link padding, compact list gaps (`padding: "0.5rem"`, `gap: "0.5rem"`) |
| md | 16px | 1rem | Card internal padding, form field gaps (`padding: "1rem"`, `gap: "1rem"`) |
| lg | 24px | 1.5rem | Section gaps, nav vertical padding (`gap: "1.5rem"`, `padding: "1.5rem 1rem"`) |
| xl | 32px | 2rem | Main content padding, major layout breathing room (`padding: "2rem"`) |
| 2xl | 48px | 3rem | Not yet used — reserve for page-level top/bottom spacing |
| 3xl | 64px | 4rem | Not yet used — reserve for full-page hero/empty state centering |

Exceptions:
- Nav sidebar fixed width: 240px (not a spacing token — structural dimension)
- Form max-width: 48rem (not a spacing token — layout constraint)
- Alert/card max-width: 36rem (used for notification banners)
- Card border-radius: 0.75rem (12px) — established pattern for all cards/sections
- Ledger table rows: no extra padding beyond sm (8px) vertical padding per cell
- Nav brand label: `font-size: 1.125rem` / `font-weight: 700` — structural exception matching existing `nav-shell.tsx`; do not change, not part of the declared type scale

## Typography

Derived from existing inline style values. Browser base font-size is 16px.

Declared type scale: 4 sizes, 2 weights.

| Role | Size | px Equivalent | Weight | Line Height | Usage |
|------|------|---------------|--------|-------------|-------|
| Body | 1rem | 16px | 400 (normal) | 1.5 | Paragraph text, table cells, form descriptions |
| Label | 0.875rem | 14px | 700 (bold) | 1.4 | Form field labels, table column headers, nav links, active nav links |
| Heading | 1.25rem | 20px | 700 (bold) | 1.2 | `<h2>` section titles (`<h2 style={{ margin: 0 }}>`) |
| Display | 1.5rem | 24px | 700 (bold) | 1.2 | `<h1>` page titles (`<h1 style={{ margin: 0 }}>`) |

Notes:
- All headings use `margin: 0` per existing pattern.
- No additional type sizes beyond these four.
- Monospace for API key display: `font-family: monospace` — applied only to key secret display cells, nowhere else.
- Nav brand label (`1.125rem` / weight 700) is a structural exception inherited from `nav-shell.tsx` — documented under Spacing Scale exceptions, not part of this type scale.

## Layout Patterns

These are prescriptive layout rules for all new Phase 9 pages — derived from existing console patterns.

### Sidebar Navigation (extended from `layout.tsx`)

New nav links added in order (below existing Dashboard and Members):

```
Dashboard
Members
---
Billing          ← new
API Keys         ← new
Analytics        ← new
Model Catalog    ← new
```

Style: identical to existing nav links — `padding: "0.5rem"`, `textDecoration: "none"`, `color: "inherit"`. Active link: `color: "#1d4ed8"` and `fontWeight: 700`.

### Page Layout

All new pages use the same max-width pattern as existing settings pages:
- Page content container: `display: "grid", gap: "1.5rem", maxWidth: "48rem"` for forms and detail pages
- Analytics and billing pages (wider tables/charts): `maxWidth: "72rem"`
- Page title block: `display: "grid", gap: "0.5rem"` containing `<h1>` + `<p>` description

### Card / Section Pattern

```
border: "1px solid #d1d5db"
borderRadius: "0.75rem"
padding: "1rem"
display: "grid"
gap: "1rem"
```

### Tab Pattern (analytics page)

Four tabs (Overview, Usage, Spend, Errors) rendered as a horizontal tab bar:
- Tab bar: `display: "flex"`, `borderBottom: "1px solid #e5e7eb"`, `gap: "0"`, `marginBottom: "1.5rem"`
- Tab item (inactive): `padding: "0.5rem 1rem"`, `color: "#6b7280"`, `borderBottom: "2px solid transparent"`
- Tab item (active): `padding: "0.5rem 1rem"`, `color: "#1d4ed8"`, `borderBottom: "2px solid #1d4ed8"`, `fontWeight: 700`
- Tab interaction: plain anchor links with URL hash or query param (`?tab=usage`)

### Table Pattern (ledger, API keys, catalog)

```
width: "100%"
borderCollapse: "collapse"

th: padding: "0.5rem 0.75rem", textAlign: "left", fontWeight: 700, fontSize: "0.875rem",
    borderBottom: "1px solid #e5e7eb", color: "#4b5563"

td: padding: "0.5rem 0.75rem", borderBottom: "1px solid #f3f4f6", fontSize: "1rem"
```

Pagination controls: "Previous" / "Next" text buttons, current page indicator, placed below table.

### Modal Pattern (checkout)

```
/* Overlay */
position: "fixed", inset: 0, backgroundColor: "rgba(0,0,0,0.4)", zIndex: 50,
display: "flex", alignItems: "center", justifyContent: "center"

/* Dialog */
backgroundColor: "#ffffff", borderRadius: "0.75rem", padding: "1.5rem",
width: "100%", maxWidth: "32rem", display: "grid", gap: "1rem"
```

Modal is a `"use client"` component. Opened by "Buy Credits" button in billing overview. Closed by "Keep balance" button or clicking the overlay.

### Chart Container Pattern (analytics)

```
/* Server component passes data as props to chart client component */
/* Chart wrapper */
"use client"
backgroundColor: "#f9fafb"
borderRadius: "0.75rem"
padding: "1rem"
marginBottom: "1.5rem"
height: 300px (fixed via ResponsiveContainer height={300})
```

## Interaction States Required

All interactive elements must implement these states (inline style changes only — no CSS classes):

| State | Visual Treatment |
|-------|-----------------|
| Default button (primary) | `background: "#1d4ed8"`, `color: "#ffffff"`, `padding: "0.5rem 1rem"`, `borderRadius: "0.375rem"`, `border: "none"`, `fontWeight: 700, cursor: "pointer"` |
| Hover button (primary) | `background: "#1e40af"` (slightly darker) — applied via `onMouseEnter`/`onMouseLeave` state |
| Disabled button | `background: "#9ca3af"`, `cursor: "not-allowed"`, `opacity: 0.7` |
| Destructive button | `background: "#dc2626"`, `color: "#ffffff"` — same sizing as primary |
| Secondary/ghost button | `background: "transparent"`, `color: "#1d4ed8"`, `border: "1px solid #1d4ed8"` |
| Loading inline | Replace button label text with "Loading..." — do not add spinner library |
| Form field | `border: "1px solid #d1d5db"`, `borderRadius: "0.375rem"`, `padding: "0.5rem"`, `width: "100%"` |
| Form field focus | `outline: "2px solid #1d4ed8"`, `outlineOffset: "2px"` — via `onFocus`/`onBlur` state |
| Form field error | `border: "1px solid #dc2626"` + error text below in `#dc2626` at 0.875rem |
| Table row hover | `background: "#f9fafb"` — via `onMouseEnter`/`onMouseLeave` |
