Create the immutable Hive Credit ledger foundation, idempotent posting model, and current-account balance read model for Phase 3.

Output: Redis plumbing for idempotency, a Supabase migration for ledger tables, a `ledger` package, and authenticated current-account balance/ledger read APIs.

1. Create `apps/control-plane/internal/ledger/types.go` with:
   - `type EntryType string`
   - constants `EntryTypeGrant`, `EntryTypeAdjustment`, `EntryTypeReservationHold`, `EntryTypeReservationRelease`, `EntryTypeUsageCharge`, `EntryTypeRefund`
   - `type LedgerEntry`
   - `type BalanceSummary` with fields `PostedCredits int64`, `ReservedCredits int64`, and `AvailableCredits int64`
   - `type PostEntryInput`

2. Create `apps/control-plane/internal/ledger/repository.go` with a pgx-backed repository that exposes:
   - `PostEntry(ctx context.Context, accountID uuid.UUID, input PostEntryInput) (LedgerEntry, error)`
   - `GetBalance(ctx context.Context, accountID uuid.UUID) (BalanceSummary, error)`
   - `ListEntries(ctx context.Context, accountID uuid.UUID, limit int) ([]LedgerEntry, error)`
   - `PostEntry` must first insert into `public.credit_idempotency_keys` and return the existing ledger entry when the `(account_id, operation_type, idempotency_key)` key already exists
   - `GetBalance` must compute:
     - `PostedCredits` as the sum of `credits_delta` where `entry_type in ('grant', 'adjustment', 'usage_charge', 'refund')`
     - `ReservedCredits` as the absolute net of `reservation_hold` plus `reservation_release`
     - `AvailableCredits` as `PostedCredits - ReservedCredits`

3. Create `apps/control-plane/internal/ledger/service.go` with:
   - `func NewService(repo Repository) *Service`
   - `func (s *Service) GetBalance(ctx context.Context, accountID uuid.UUID) (BalanceSummary, error)`
   - `func (s *Service) ListEntries(ctx context.Context, accountID uuid.UUID, limit int) ([]LedgerEntry, error)`
   - `func (s *Service) GrantCredits(ctx context.Context, accountID uuid.UUID, idempotencyKey string, credits int64, metadata map[string]any) (LedgerEntry, error)`
   - `func (s *Service) AdjustCredits(ctx context.Context, accountID uuid.UUID, idempotencyKey string, creditsDelta int64, metadata map[string]any) (LedgerEntry, error)`
   - validation rules:
     - grants require `credits > 0`
     - adjustments require `creditsDelta != 0`
     - blank idempotency keys are rejected

4. Create `apps/control-plane/internal/ledger/http.go` following the `profiles.Handler` pattern:
   - handler constructor `NewHandler(svc *Service, accountsSvc *accounts.Service) *Handler`
   - `GET /api/v1/accounts/current/credits/balance`
   - `GET /api/v1/accounts/current/credits/ledger`
   - both routes resolve the current account from the authenticated viewer context instead of trusting request body account IDs
   - `GET /ledger` accepts `limit` query param and defaults to `20`
