## Objective
Close the remaining Phase 13 DIFF-01 gap by seeding static no-dispatch headers inside `v1Plugin` for validation and other plugin-generated pre-dispatch errors, then lock that behavior with direct validation regressions on the real plugin surface.

## Context
- `13-VERIFICATION.md` proved the only remaining blocker is plugin-generated validation/pre-handler errors that run before route handlers can call `setNoDispatchDiffHeaders(reply)`.
- `apps/api/src/routes/diff-headers.ts` already provides the canonical no-dispatch header helper and uses the repo-safe sequential `reply.header(...)` pattern.
- `apps/api/src/routes/v1-plugin.ts` already owns the scoped `/v1/*` error boundary and `x-request-id`; this gap should be closed there instead of broadening `sendApiError()`.
- `apps/api/test/routes/typebox-validation.test.ts` already boots the real `v1Plugin` with lightweight mock services, so it is the correct place to assert the missing DIFF-header contract.
- Preserve the deferred boundary from `13-CONTEXT.md`: do not redesign generic unknown-route behavior or change non-`/v1/*` error helpers. Only add the static no-dispatch headers to plugin-generated responses.
