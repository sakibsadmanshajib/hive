## Objective
Remove the embeddings regression blind spot by making the SDK success case use the real catalog and runtime wiring instead of the helper's mock catalog, then prove the OpenAI SDK succeeds with the public model id `text-embedding-3-small`.

## Context
- `12-CONTEXT.md` makes the official SDK success path the primary acceptance boundary and explicitly rejects mock-only evidence.
- `12-RESEARCH.md` and the milestone audit both call out the masking point: `apps/api/test/helpers/test-app.ts` directly lists `text-embedding-3-small`, so the existing SDK regression never exercised the production catalog.
- `11-01-SUMMARY.md` shows the regression suite already uses a shared mock app for many success/error paths. Preserve that where it still adds value, but do not let it remain the embeddings acceptance surface.
- `12-01` will already have made `text-embedding-3-small` the canonical public catalog id and fixed the provider/public model split. This plan only changes the regression harness so that those runtime behaviors are actually exercised by the SDK test.
