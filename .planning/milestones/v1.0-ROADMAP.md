# Milestone v1.0: OpenAI API Compliance

**Status:** SHIPPED 2026-03-22
**Phases:** 1-13
**Total Plans:** 25

## Overview

This milestone turned Hive's public `/v1/*` surface into a drop-in OpenAI-compatible API that is verifiable with the official `openai` Node SDK, route-level regressions, and Docker-local build evidence.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Keep `/v1/*` behavior behind the scoped `v1Plugin` so OpenAI-compatible behavior can evolve without leaking into the web pipeline.
- Keep public model ids provider-agnostic and expose provider-specific ids only through DIFF headers.
- Use real OpenAI SDK regressions plus Docker-local builds as milestone-level verification, not unit tests alone.
- Seed static no-dispatch DIFF headers in route and plugin boundaries rather than broadening shared non-`/v1/*` error helpers.

**Issues Resolved:**

- OpenAI error-envelope incompatibilities across `/v1/*`
- Missing auth and DIFF headers on models routes
- Broken real-runtime embeddings alias behavior for `text-embedding-3-small`
- Missing DIFF headers on error, validation, and stub responses

**Issues Deferred:**

- Deeper downstream adoption of generated OpenAI spec types
- Stronger live route assertions for `x-request-id`

**Technical Debt Incurred:**

- Phase 12 summary/tracking artifacts had to be recovered after execution because the original run did not finish artifact writing
