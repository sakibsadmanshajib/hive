- The regression harness mounts `v1Plugin` with mocks instead of booting the full runtime, which let the embeddings alias gap escape.

## Conclusion

Milestone `v1.0` is **not ready for archival**.

Two requirement-level blockers remain:
1. `DIFF-03` is unsatisfied because the real embeddings runtime does not accept the standard SDK-facing model name `text-embedding-3-small`.
2. `DIFF-01` is unsatisfied because DIFF headers are not present on non-model error paths and unsupported endpoint stubs.

Those blockers break two end-to-end flows and force the milestone audit status to `gaps_found`.
