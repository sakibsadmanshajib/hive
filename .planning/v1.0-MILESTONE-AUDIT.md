## Unsatisfied Requirements

### FOUND-02 — Bearer Auth on Models Routes

**Severity: Medium**

Phase 10 (`10-models-route-compliance`) was created to fix this gap but was never executed — the directory is empty. `GET /v1/models` and `GET /v1/models/:model` in `apps/api/src/routes/models.ts` (lines 8, 15) have no `requireV1ApiPrincipal` call. Any unauthenticated HTTP client gets 200 + the full model catalog.

OpenAI requires auth on models routes — SDK clients with invalid keys calling `client.models.list()` receive a model list instead of `openai.AuthenticationError`, breaking drop-in compatibility.

**Fix:** Execute Phase 10 — add `requireV1ApiPrincipal(request, reply)` at the top of both route handlers in `models.ts`.

---

### DIFF-01 — AI Routing Headers on Models Routes

**Severity: Low**

Phase 10 was also responsible for this fix and was never executed. `apps/api/src/routes/models.ts` sets none of the 4 AI routing headers (`x-hive-provider`, `x-hive-model`, `x-hive-latency-ms`, `x-actual-credits`). The requirement says "all /v1/* endpoints". Semantically N/A for a catalog read (no AI dispatch, no credit charge), but technically violates the requirement as written.

**Fix:** Execute Phase 10 — either scope the requirement to AI-dispatching routes, or add static/zero values to models handlers.
