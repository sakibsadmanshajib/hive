# v1.1 Master Plan v3 — All 15 Phases

## v3 Pivot Summary (2026-04-25)

Phase 19 research (see `.planning/phases/19-chat-app-fork-strip/RESEARCH.md`) found two blockers in v2's Lobe Chat plan: (1) Lobe Chat ships under "LobeHub Community License" (Apache-2.0 + §1.b commercial-derivative clause) — Hive-rebranded fork is a derivative requiring paid commercial license; (2) upstream Lobe is NOT Cloudflare Workers compatible (maintainer confirmed in issue #4241; sharp/pg/ws/pdf-parse/mammoth deps are Node-only).

**Decisions locked** (see `.planning/v1.1-chatapp/LICENSE-DECISION.md`):

- **Base = LibreChat (MIT).** Replaces Lobe Chat. MIT clears derivative-work issues. Adds MongoDB to infra. Optional Python RAG API sidecar.
- **Hosting = chat-app on OCI containers, web-console on Cloudflare Workers (unchanged).** Workers is dropped as a chat-app target. DNS + TLS via Cloudflare in front of OCI.
- All Track B phase summaries re-ground on LibreChat (Phase 19, 20, 22, 24 most affected; see §Track B).

The Lobe Chat research artifact remains on disk as a record of the rejected alternative.

Compact phase plans. Each phase will spawn a full GSD `PLAN.md` via `/gsd:plan-phase` before execution. This document is the milestone source of truth.

## Tier Model

| Tier | Email | Phone | Chat | File Upload | Rate Limit | Credit |
|------|-------|-------|------|-------------|-----------|--------|
| **Guest** | none | none | yes | no | tight (e.g. 10 msg/day, 2k tokens/msg) | none |
| **Unverified** | yes (unverified) or verified-only | unverified | yes | no | medium (e.g. 50 msg/day, 4k tokens/msg) | none |
| **Verified** | verified | verified | yes | **yes** | normal free tier (e.g. 200 msg/day, 8k tokens/msg, 5 files/day, 10 MB/file) | none |
| **Credited** | verified | verified | yes | yes | per-credit consumption | discretionary, owner-granted |

Numbers above are placeholders — finalized in Phase 21 PLAN.md. All limits enforced via existing Hive rate-limit infra (Phase 12).

## Architecture (Track B — v3, regrounded on LibreChat + OCI)

```
hive/apps/chat-app                ← new, fork of LibreChat (pinned tag)
  ├── auth: Supabase (shared with web-console SSO)
  ├── default OpenAI-compatible provider → hive edge-api (http://edge-api:8080/v1)
  ├── primary DB: MongoDB (LibreChat-native — Atlas M0 vs OCI self-host decided in Phase 19)
  ├── secondary DB: Postgres (chat_app schema in same Supabase project) for tier/credit/referral state
  ├── files: Supabase Storage (hive-files bucket, reused) — verified-tier+ only
  ├── embeddings: hive edge-api /v1/embeddings → Supabase pgvector (HNSW index)
  ├── billing: per-message metering via existing prepaid ledger (credited tier)
  ├── rate limits: per-tier from Phase 12 limiter
  ├── i18n: bn-BD primary, en-US secondary, picker on first run
  ├── invite: invite codes + referral tracking
  └── deploy: OCI container instance(s); Cloudflare DNS/TLS in front (Workers NOT a target)
```

## Phase 12 — KEY-05 Hot-Path Rate Limiting [A]

**Goal:** Per-API-key rate limits enforced in `edge-api` hot path. **Tiered limits** for chat-app users (per Tier Model) layered on top.

**Scope:**
- Token-bucket rate limiter in Redis (existing infra).
- Limits configurable per-key in control-plane (DB column on `api_keys`).

- Standard `X-RateLimit-*` response headers.
- 429 with `Retry-After` on overflow.

**Tasks:**
1. Migration: `api_keys.rate_limit_rpm`, `rate_limit_tpm`, `tier_overrides` columns.
2. control-plane: CRUD endpoints + console UI to set per-key + tier limits.
3. edge-api: middleware checks Redis token bucket before LiteLLM dispatch. Tier resolved from auth context.
4. Tests: unit (limiter math), integration (concurrent burst, tier enforcement), load test.
5. Metrics: `rate_limit_exceeded_total{tier=...}` Prometheus counter.

**Deps:** none. **Success:** verified user chat exceeds tier limit → 429 with Retry-After. Guest user upload attempt → 403 (handled in chat-app, not rate limit).

**Risk:** medium — hot path latency. Budget: <2ms p99 added.
