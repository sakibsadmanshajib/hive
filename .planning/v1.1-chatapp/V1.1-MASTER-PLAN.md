# v1.1 Master Plan v4 - All 16 Phases

## v4 Pivot Summary (2026-05-17)

Phase 19 planning moved Track B from a vendored LibreChat fork to an upstream Open WebUI container. Open WebUI is consumed as an image, not forked. Hive customisation lands through environment, Supabase OIDC, pipeline filters, Caddy path blocking, and our existing Go services.

The old LibreChat planning files remain only as historical records:

- `.planning/v1.1-chatapp/LICENSE-DECISION.DEPRECATED.md`
- `.planning/v1.1-chatapp/LIBRECHAT-VERSION.DEPRECATED.md`
- `.planning/v1.1-chatapp/LIBRECHAT-UPGRADE-PLAYBOOK.DEPRECATED.md`
- `.planning/phases/19-chat-app-fork-strip/`

The non-deprecated `LICENSE-DECISION.md` is now the active Open WebUI license risk note. The non-deprecated `LIBRECHAT-*` files are short pointers retained so archived links do not break.

Phase 26 is a scope addition for first-party web search. It stays numbered 26 to avoid renumbering Phase 23-25 references, but its execution slot is after Phase 21 credit/quota primitives and before Phase 24 self-host packaging and Phase 25 payment/cutover work if web search is still in the v1.1 launch gate.

Legend: **A** = Track A API hardening. **B** = Track B Hive Chat.

## Tier Model

| Tier | Identity | Chat | Upload / RAG | Web search | Credit |
|------|----------|------|--------------|------------|--------|
| Guest | no verified identity | yes, tight limits | no | no | none |
| Unverified | account exists, verification incomplete | yes, medium limits | no | no | none |
| Verified | required verification complete | yes, normal free quota | personal RAG allowed | daily quota allowed | none by default |
| Credited | verified plus tenant/user credit allocation | yes, paid/pooled usage | allowed | paid/pooled over quota | tenant pool / grant |

Numbers are placeholders until Phase 21. All limits reuse the Phase 12 hot-path limiter and the Phase 21 credit/quota model.

## Track B Architecture

```text
browser
  -> Open WebUI (chat + personal RAG, admin disabled)
  -> Supabase Auth OIDC (custom_access_token_hook adds tenant claims)
  -> OWUI pipeline filter forwards the Supabase JWT
  -> edge-api validates JWT / API key and dispatches to LiteLLM
  -> control-plane owns tenant settings, provider catalog, credits, audit, and admin APIs
  -> Supabase Postgres stores tenants, audit_log, llm_traces, RAG metadata, and credit state
  -> SearXNG serves web search on the internal Docker network only
```

Open WebUI uses native pgvector-backed personal RAG. Shared tenant knowledge bases are Hive-owned and land in Phase 22 through control-plane ingestion and edge-api retrieval injection.

## Phase Summaries

### Phase 11 - Compliance, Verification & Artifact Cleanup [A]

Close v1.0 traceability gaps, add formal verification artifacts, and clean stale planning state.

### Phase 12 - KEY-05 Hot-Path Rate Limiting [A]

Per-key and tier-scoped RPM/TPM limits in edge-api with standard rate-limit headers.

### Phase 13 - Console Integration Fixes [A]

Audit-first web-console integration sweep, BDT-only billing surface, and strict TypeScript cleanup.

### Phase 14 - Payments, Invoicing & Budget Integration [A]

Invoice rows, budget threshold enforcement, spend alerts, and owner-discretionary credit grants.

### Phase 15 - Batch Local Executor [A]

Local fan-out executor for batch success-path settlement where upstream providers do not expose native batch APIs.

### Phase 16 - Capability Columns Fix [A]

Remove the dead runtime DDL path and make migrations authoritative for provider capability columns.

### Phase 17 - FX/USD Zero-Leak [A]

Zero customer-visible USD, FX rates, or exchange language across control-plane, web-console, PDFs, metadata maps, and future chat surfaces.

### Phase 18 - RBAC Matrix [A]

Verification-aware permission matrix used by control-plane handlers and web-console gates.

### Phase 19 - Foundation Slice [B]

Tenant settings, tenants/users schema, Supabase custom access token hook, JWT/API-key selector, Open WebUI compose service, Caddy admin stripping, chat happy path, `audit_log`, `audit_outbox`, `llm_traces`, and SOC 2 evidence generation.

### Phase 20 - Provider Catalog [B]

Hybrid provider catalog. Stock providers are migration-seeded. Custom providers are DB-managed. control-plane regenerates LiteLLM config and reloads it. Open WebUI model visibility follows tenant/provider policy.

### Phase 21 - Credit and Quota Engine [B]

Tenant credit pool, per-user soft caps, monthly grants, owner top-ups, extra-usage top-ups, and bucket rate limits. This phase defines the tier/credit primitives consumed by RAG, admin pages, and web search.

### Phase 22 - Shared Knowledge-Base RAG [B]

Admin-managed tenant KBs. control-plane chunks and embeds documents through LiteLLM; edge-api injects retrieval context at chat time. Personal RAG remains Open WebUI native.

### Phase 23 - Admin Console Pages [B]

web-console pages for tenant settings, provider management, users/roles, audit viewer, credit controls, and operational status.

### Phase 24 - EnterpriseEdge Self-Host Packaging [B]

Customer self-host bootstrap, single-tenant defaults, env templates, docs, backup/restore, optional sink configuration, and optional SearXNG inclusion if Phase 26 remains in v1.1.

### Phase 25 - Payments Tenant-Gating and Hive Cloud Cutover [B]

Gate existing Stripe, bKash, and SSLCommerz flows behind tenant settings, re-audit customer-visible billing/chat surfaces, and cut Hive Cloud to the v1.1 stack.

### Phase 26 - Web Search Tool [B]

Self-host SearXNG plus `POST /v1/tools/web_search`, `GET /v1/tools` catalog advertisement, OpenAPI overlay, OWUI native web-search wiring, tier gates, rate limits, BDT ledger integration, provider-blind error sanitisation, and SDK/E2E coverage.

## Out of Scope (v1.2+)

- Search verticals beyond general web results (image, news, video).
- Per-query LLM result ranking.
- Long-term result caching.
- Ads tier.
- Voice features.
- Regional rollout beyond BD.
