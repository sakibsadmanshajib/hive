## Chunk 2: Implement Real Batch Submission

### Task 3: Add the control-plane batch submitter

**Files:**
- Create: `apps/control-plane/internal/batchstore/submitter.go`
- Modify: `apps/control-plane/internal/filestore/http.go`
- Modify: `apps/control-plane/cmd/server/main.go`

- [ ] **Step 1: Implement the submitter**

Add a submitter that:
1. loads the input file from storage,
2. selects a batch-capable route,
3. rewrites JSONL `body.model` to the selected LiteLLM model,
4. uploads the rewritten file to LiteLLM,
5. creates the upstream batch,
6. stores `upstream_batch_id` and progress fields,
7. enqueues the first `batch:poll` task.

- [ ] **Step 2: Wire the submitter into the internal create path**

`POST /internal/batches/create` should persist the row, submit the upstream batch, and return the refreshed batch object. On immediate submission failure, it should release the reservation and mark the batch failed.

- [ ] **Step 3: Configure LiteLLM managed file support**

Update `deploy/litellm/config.yaml` with the `files_settings` needed for `/v1/files` batch uploads in this environment.

- [ ] **Step 4: Run targeted tests to verify green**

Run: `go test ./apps/control-plane/internal/batchstore ./apps/control-plane/internal/filestore -count=1`
