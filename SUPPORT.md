# Support

## Where to Ask for Help

Use the right channel for the right kind of request:

- bug reports: open a GitHub issue with reproduction steps, expected behavior, and actual behavior
- feature requests: open a GitHub issue describing the problem, proposed outcome, and acceptance criteria
- contribution questions: open a GitHub issue or draft PR and reference `CONTRIBUTING.md`
- security issues: follow `SECURITY.md` and do not post them publicly

## Before Opening an Issue

- read `README.md`
- check `docs/README.md` for architecture and runbooks
- search existing issues and pull requests for duplicates
- include the exact command, route, file path, or environment detail relevant to the problem

## What to Include

Helpful reports usually contain:

- the affected area, such as `apps/api`, `apps/web`, `supabase/migrations`, or `docs`
- reproduction steps
- expected result
- actual result
- logs or screenshots when relevant
- verification command output when relevant

## Support Boundaries

Hive is maintained as an open-source project. Support is best-effort.

Maintainers may prioritize:

- security issues
- billing correctness
- production-safety regressions
- public beta blockers

General debugging, local environment problems, or custom deployment help may receive limited support.

## Conduct

All support interactions are covered by `CODE_OF_CONDUCT.md`.
