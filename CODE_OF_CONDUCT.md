# Code of Conduct

## Our Standard

Hive aims to be a respectful, technically rigorous, and collaborative project.

Participants are expected to:

- act professionally and respectfully
- focus on evidence, code, docs, and behavior rather than personal attacks
- welcome good-faith questions and review feedback
- avoid harassment, discrimination, intimidation, or abusive conduct
- respect privacy and avoid posting sensitive information without permission

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassing, insulting, or demeaning comments
- personal or political attacks unrelated to the project work
- publishing private information, credentials, or security-sensitive details
- disruptive trolling, bad-faith review behavior, or repeated hostility
- any conduct that creates an unsafe or exclusionary environment

## Scope

This Code of Conduct applies to:

- GitHub issues, pull requests, and reviews
- project documentation and discussions
- other project-managed communication spaces used for Hive collaboration

## Reporting

If you experience or witness unacceptable behavior, report it privately rather than escalating publicly.

Primary reporting path:

- contact the repository owner privately through GitHub at `https://github.com/sakibsadmanshajib`

When reporting, include:

- what happened
- where it happened
- any relevant links or screenshots
- whether there is an immediate safety or confidentiality concern

## Enforcement

Maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that violate this Code of Conduct.

Possible responses include:

- warning the contributor
- limiting participation in project spaces
- closing or restricting discussion threads
- removing contribution privileges

## Project Values

Hive favors clear reasoning, production-safe changes, and respectful disagreement. Strong technical review is welcome; hostile behavior is not.
